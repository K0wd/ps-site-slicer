Here's a summary of the changes made to `purchasing-tracker.steps.ts`:

**Fixes applied:**

1. **Set filter "Select All" handling** — ag-Grid set filters start with all options checked. The rewrite uncheck "(Select All)" first via `ag-set-filter-select-all` checkbox, then checks the desired option. Applied to `I select {string} from the set filter`, `I select a known approver`, and `I select the first available set filter option`.

2. **Filter menu button hover** — Added `headerCell.hover()` before clicking the menu button, since ag-Grid hides the hamburger icon until hover. Also added visibility assertion on the menu button before clicking.

3. **Filter tab navigation** — After opening the ag-Grid menu, clicks the filter tab (icon-filter) if the menu has tabs, since ag-Grid shows columns/filter/general tabs.

4. **Proper "clear the active filter"** — Instead of just pressing Escape, now detects whether it's a text filter (clears the input) or set filter (re-checks "Select All"), then closes the popup.

5. **Unfiltered state comparison** — Captures initial row count when landing on the page, then `the table should return to unfiltered state` verifies the count is back to the original.

6. **Division no-filter check** — Now hovers the header cell first before asserting the menu button doesn't exist, avoiding false positives from hidden-by-default buttons.

7. **Close popup before assertions** — `the table should show filtered rows` and `...or zero results` now close any open popup first so the grid renders updated data.

8. **Set filter checkbox targeting** — Uses `AG_SET_FILTER_CHECKBOX_XPATH_FN` (the checkbox container) instead of clicking the text span, which is more reliable for toggling.

9. **Removed `Given` import** — Only `When` and `Then` are needed since `Given` steps are defined in other files.
