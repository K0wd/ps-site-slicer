KEY FINDINGS
------------
1. EXPORT MECHANISM: The app uses a server-side export with a modal progress dialog
   ("Exporting... NN%"). This dialog is a blocking overlay that prevents ALL other UI
   interactions while active. The export can take several minutes for the full dataset.
   This is critical for test automation — exports must complete (or be dismissed) before
   any subsequent filter tests can run.

2. APPROVAL COLUMN IS DROPDOWN: Confirmed via EC-06 screenshot — the Approval filter
   uses a set filter (checkbox dropdown) with values including: (Select All), Blanks, No,
   Override, Pending Approval. This matches the expected behavior from SM-754/SM-829 fix.

3. DIVISION FILTER REMOVED: Confirmed — no filter menu button on Division column header.

4. DATE FILTER PRESENT: Request Date filter menu opened and condition was applied in TC-03.

5. EXPORT OVERLAY PERSISTENCE: The export overlay persisted across ~20 test cases. This
   suggests either: (a) the export takes very long on the test server, or (b) the overlay
   is not auto-dismissing after completion. This should be investigated for SM-756.

RECOMMENDATIONS
---------------
1. Re-run tests sequentially, waiting for any export overlay to fully dismiss before
   proceeding to the next test. Add explicit waits for the export dialog to disappear.
2. For ID column (TC-01/02): investigate the ag-Grid cell rendering — the text may be
   in a child element like .ag-cell-value rather than directly in the col-id div.
3. The export mechanism needs dedicated handling — do NOT use browser download events.
   Instead, wait for the "Exporting..." overlay to appear, track progress, wait for it
   to disappear, then check for the download.
4. Consider running filter-only tests separately from export tests to avoid the overlay
   blocking issue.
