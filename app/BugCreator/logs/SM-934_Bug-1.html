Description (paste into Jira Description field)

Issue observed: When editing a vendor and the browser window is resized to a narrow width, the action buttons in the edit vendor view become tightly packed and begin overlapping each other, making them difficult or impossible to click.

Expected: Buttons should remain accessible and non-overlapping at all reasonable window widths. They should either wrap to a new line, stack vertically, or otherwise adapt gracefully to a constrained viewport.

Steps to reproduce:

  1. Log in to Site Manager and navigate to the Vendor Admin section.
  2. Open an existing vendor for editing.
  3. Resize the browser window to a narrow width (e.g. approximately 1024px or below — TBD exact breakpoint).
  4. Observe the action buttons in the edit vendor view.

Notes: This is a responsive layout issue. The breakpoint at which overlap occurs has not yet been documented — record the exact window width when reproducing. A screenshot showing the overlap should be attached.
